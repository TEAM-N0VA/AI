# 🩸 밀당 - AI

임신성 당뇨 산모를 위한 **YOLO 기반 식단 자동 기록 및 RAG 기반 식단 코칭 서비스 밀당의 AI 서버 리포지토리**입니다.
본 프로젝트는 음식 이미지 객체 인식, 임신성 당뇨 맞춤형 챗봇, 사용자 맞춤 식당 추천 기능을 제공하기 위해 FastAPI 기반으로 개발되었습니다.

AI 서버는 프론트엔드 및 백엔드 서버와 연동되어 사용자의 식단 기록 피로도를 낮추고, 산모가 혈당과 식단을 보다 안전하고 지속적으로 관리할 수 있도록 지원합니다.

---

## 📑 프로젝트 개요

밀당은 매 끼니마다 엄격한 식단 통제와 혈당 관리가 필요한 임신성 당뇨 산모를 위한 AI 기반 지능형 헬스케어 서비스입니다.
산모는 음식 사진을 업로드하여 식단을 자동으로 기록할 수 있으며, 혈당 수치나 식단 관련 질문이 있을 때 챗봇을 통해 맞춤형 답변을 받을 수 있습니다.

AI 서버는 다음과 같은 핵심 기능을 제공합니다.

* 📸 **YOLOv8 기반 음식 객체 인식**을 통해 사진 한 장으로 음식 탐지 및 식단 기록 지원
* 💬 **RAG 기반 챗봇**을 통해 임신성 당뇨 산모에게 적합한 식단 및 혈당 관리 답변 제공
* 🧠 사용자 질문에서 혈당 수치, 식사 시간, 음식 키워드를 분석하여 상황에 맞는 응답 생성
* 📍 사용자 위치 기반 맞춤 식당 추천 기능 제공
* 🔗 프론트엔드 및 백엔드 서버와 API 방식으로 연동하여 모바일 앱 내 AI 기능 지원

본 AI 기능은 의료적 진단이나 처방을 대체하는 것이 아니라, 임신성 당뇨 산모의 생활 관리와 식단 기록을 돕기 위한 보조 서비스입니다.

---

## 🧩 주요 기능

### 1. 음식 이미지 인식 기능

사용자가 식단 사진을 업로드하면 YOLOv8 모델이 이미지 내 음식 객체를 탐지합니다.
탐지된 음식명은 백엔드의 음식 및 영양성분 데이터와 연결되어 식단 기록에 활용됩니다.

#### 처리 흐름

1. 사용자가 프론트엔드에서 음식 사진 업로드
2. `/meals` API를 통해 AI 서버로 이미지 전달
3. YOLOv8 모델이 이미지 내 음식 객체 탐지
4. 탐지된 음식명과 신뢰도 반환
5. 백엔드와 연동하여 영양성분 및 식단 기록에 활용

#### 응답 예시

```json
{
  "detected_foods": [
    {
      "name": "현미밥",
      "confidence": 0.91
    },
    {
      "name": "닭가슴살",
      "confidence": 0.87
    }
  ]
}
```

---

### 2. RAG 기반 식단 코칭 챗봇

사용자가 혈당, 식단, 간식, 외식 등과 관련된 질문을 입력하면 AI 서버는 질문을 분석하고, 임신성 당뇨 관련 문서를 기반으로 답변을 생성합니다.

#### 처리 흐름

1. 사용자가 챗봇 화면에서 질문 입력
2. `/chat` API를 통해 AI 서버로 질문 전달
3. 질문 전처리 및 의도 분석

   * 혈당 수치 추출
   * 식사 시간 추출
   * 음식 키워드 추출
   * 질문 유형 분류
4. 벡터 DB에서 관련 문서 검색
5. LLM을 통해 사용자 상황에 맞는 답변 생성
6. 프론트엔드로 답변 반환

#### 질문 예시

```text
식후 1시간 혈당이 145인데 괜찮나요?
```

#### 답변 예시

```text
식후 1시간 혈당 145mg/dL은 다소 주의가 필요한 수치입니다.
현재 상태에서는 추가 간식 섭취를 피하고, 가능하다면 10~20분 정도 가볍게 걷는 것이 도움이 될 수 있습니다.
반복적으로 높은 혈당이 나타난다면 담당 의료진과 상담하는 것을 권장합니다.
```

---

### 3. 맞춤 식당 추천 기능

사용자의 위치와 임신성 당뇨 식단 기준을 바탕으로 주변 식당 후보를 추천합니다.
외부 지도 및 식당 정보 API와 연동하여 주변 식당 정보를 가져오고, 산모가 비교적 안전하게 선택할 수 있는 메뉴 중심으로 결과를 제공합니다.

## 🛠 오픈 소스 프레임워크

본 프로젝트는 Python 및 FastAPI 프레임워크를 기반으로 개발되었습니다.

* **Python**: v3.10.x 권장
* **FastAPI**: AI API 서버 구현
* **Uvicorn**: FastAPI 서버 실행
* **YOLOv8**: 음식 이미지 객체 인식
* **LangChain**: RAG 기반 챗봇 파이프라인 구성
* **ChromaDB**: 임신성 당뇨 관련 문서 벡터 DB 저장 및 검색
* **LLM API**: 챗봇 답변 생성
* **python-dotenv**: 환경 변수 관리

## 🚀 설치 및 실행 방법

프로젝트를 클론하여 로컬에서 AI 서버를 설치하고 실행할 수 있도록 단계별로 안내합니다.

### 1️⃣ 실행 환경 확인

클론을 진행하기 전, PC에 아래 환경이 세팅되어 있어야 합니다.

* Python v3.10.x 권장
* pip
* Git

환경이 설치되어 있는지 확인하려면 아래 명령어를 입력합니다.

```bash
python --version
pip --version
git --version
```

---

### 2️⃣ 저장소 클론

명령 프롬프트 또는 터미널을 실행한 후, 아래 명령어를 입력하여 소스코드를 내 컴퓨터에 복사하고 해당 디렉토리로 이동합니다.

```bash
git clone https://github.com/TEAM-N0VA/AI.git
cd AI
```

---

### 3️⃣ 가상환경 생성 및 활성화

```bash
python -m venv venv
```

#### Windows


#### Mac/Linux

```bash
source venv/bin/activate
```

가상환경이 정상적으로 활성화되면 터미널 앞에 `(venv)` 표시가 나타납니다.

### 4️⃣ 패키지 설치

프로젝트 루트 디렉토리에서 아래 명령어를 실행하여 필요한 Python 패키지를 설치합니다.

```bash
pip install -r requirements.txt
```

---

### 5️⃣ 환경 변수 및 모델 파일 준비

AI 서버 실행을 위해 `.env` 파일을 생성하고 필요한 값을 입력합니다.
또한 음식 이미지 인식 기능을 사용하기 위해 `model/` 폴더 안에 YOLO 모델 파일을 배치해야 합니다.

```bash
cp .env.example .env
```

`.env` 파일 예시는 다음과 같습니다.

```env
# SERVER SETTINGS
DEBUG_MODE=true
PORT=8000

# AI MODEL PATHS
YOLO_MODEL_PATH=./model/food_3_best.pt

# LLM API
OPEN_API_KEY={직접 발급받은 API 키}

# CORS SETTINGS
ALLOW_ORIGINS={FE/BE 서버 주소}

# VECTOR DB
CHROMA_DB_PATH=./vector_db
```

모델 파일은 아래 위치에 배치합니다.

```bash
AI/
└── model/
    └── food_3_best.pt
```

> API Key, 서버 주소, 모델 파일 경로 등은 팀 내부 노션 또는 카카오톡 공지를 참고하여 입력합니다.
> `.env` 파일에는 민감한 정보가 포함되므로 GitHub에 업로드하지 않습니다.

## 🔗 API 엔드포인트

### 1. 서버 상태 확인

#### `GET /`

AI 서버가 정상적으로 실행 중인지 확인하는 엔드포인트입니다.

#### 응답 예시

```json
{
  "message": "Mealdang AI server is running."
}
```

---

### 2. 음식 이미지 인식

#### `POST /meals`

사용자가 업로드한 음식 이미지를 분석하여 탐지된 음식 정보를 반환합니다.

#### 요청 형식

* `multipart/form-data`
* 이미지 파일 업로드

#### 응답 예시

```json
{
  "detected_foods": [
    {
      "name": "현미밥",
      "confidence": 0.91
    },
    {
      "name": "계란찜",
      "confidence": 0.84
    }
  ]
}
```

---

### 3. 챗봇 대화 생성

#### `POST /chat`

사용자의 질문을 받아 임신성 당뇨 식단 관리에 적합한 답변을 생성합니다.

#### 요청 예시

```json
{
  "question": "식후 1시간 혈당이 145인데 괜찮나요?"
}
```

#### 응답 예시

```json
{
  "answer": "식후 1시간 혈당 145mg/dL은 다소 주의가 필요한 수치입니다. 추가 간식 섭취는 피하고, 식후 가벼운 걷기를 권장합니다."
}
```

---

### 4. 맞춤 식당 추천

#### `POST /recommend`

사용자의 위치와 식단 조건을 바탕으로 주변 식당 추천 결과를 반환합니다.

#### 요청 예시

```json
{
  "latitude": 37.5665,
  "longitude": 126.9780,
  "keyword": "한식"
}
```

#### 응답 예시

```json
{
  "restaurants": [
    {
      "name": "건강한 한식당",
      "address": "서울특별시 ...",
      "category": "한식",
      "reason": "비교적 담백한 메뉴 선택이 가능하여 임신성 당뇨 식단 관리에 적합합니다."
    }
  ]
}
```

## 🧪 테스트 방법

서버 실행 후 Swagger 문서에서 API를 직접 테스트할 수 있습니다.

```text
http://127.0.0.1:8000/docs
```

또는 curl 명령어로 테스트할 수 있습니다.

### 챗봇 테스트

```bash
curl -X POST "http://127.0.0.1:8000/chat" \
-H "Content-Type: application/json" \
-d '{
  "question": "바나나 먹어도 되나요?"
}'
```

### 음식 이미지 인식 테스트

```bash
curl -X POST "http://127.0.0.1:8000/meals" \
-F "file=@sample_food.jpg"
```
